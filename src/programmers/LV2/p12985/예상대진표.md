### 문제 분석 요약
- 한줄에 w개씩 n개의 택배상자가 있다
- 꺼내려고 하는 상자 번호가 주어질때 몇개의 택배 상자를 꺼내야하는지 구하는 문제

### 제약조건
N : 2^1 이상 2^20 이하인 자연수 (2의 지수 승으로 주어짐)
A, B : N 이하인 자연수 (A ≠ B)
A와 B는 서로 만나기 전까지 항상 승리한다고 가정
### 알고리즘 설계

문제의 핵심
각 라운드마다 참가자 번호가 절반으로 줄어들면서 재배정된다
두 참가자가 같은 대진에 속하는 라운드를 찾는다
#### 입력
- N: 참가자 수
- A: 첫 번째 참가자 번호
- B: 경쟁자 참가자 번호
#### 연산
- 라운드를 1부터 시작하여 반복
- 각 라운드에서 A와 B가 같은 대진에 속하는지 확인
- 같은 대진이면 현재 라운드 번호 반환
- 다르면 A와 B를 다음 라운드 번호로 갱신
- 라운드 증가 후 반복

#### 출력
- A와 B가 만나는 라운드 번호
### 시간 복잡도
- O(log N)
    - 매 라운드마다 참가자 수가 절반으로 줄어든다

### 코드
```java  
class Solution
{
    public int solution(int n, int a, int b)
    {
        int answer = 0;

        for(answer = 0; a != b; answer++){
            a = (a + 1) / 2;
            b = (b + 1) / 2;
        }

        return answer;
    }
}
```

### 알게된점
- 코드는 간단하지만 트리의 개념을 어떻게 활용할 수 있는지 생각해 볼 수 있는 문제
